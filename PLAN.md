# プロジェクト企画書：落語コンシェルジュ AI

## 1. プロジェクト概要
**サービス名：** 落語コンシェルジュ AI (Remix)
**コンセプト：** 「伝統と革新の融合」 — AIが400年の歴史を持つ落語の世界を、現代の視聴者へパーソナライズして届ける。

## 2. 開発の背景と目的
### 背景
- 落語は400年以上の歴史を持つ日本の素晴らしい伝統芸能だが、演目数が非常に多く（古典・新作合わせ数千以上）、初心者が「どれから聴けばいいか」を判断するのが難しい。
- 若年層やインバウンド層にとって、入り口となる適切なガイドラインが不足している。

### 目的
- AIを活用し、ユーザーの「今の気分」や「興味」に合わせた最適な演目を提案することで、落語への心理的ハードルを下げる。
- 演目の詳細情報（あらすじ、見どころ、難易度）を分かりやすく提供し、落語の楽しみ方を深く伝える。

## 3. ターゲットユーザー
1. **落語初心者：** 落語に興味はあるが、どの演目から聴き始めれば良いか分からない層。
2. **中級・上級者：** 自分の好みのジャンルは把握しているが、新しいジャンルや珍しい演目に出会いたい層。
3. **伝統好きの若年層：** デジタルツールを使いこなしつつ、日本の文化に触れたい層。

## 4. 主な機能
- **AIパーソナライズ提案：** ユーザーのキーワード入力に基づき、Gemini AIが最適な3席を厳選してレコメンド。
- **演目検索機能：** 特定のキーワードやジャンルから演目を検索。
- **詳細解説画面：**
  - **あらすじ：** 200〜400文字の丁寧なストーリー解説。
  - **難易度表示：** 初級・中級・上級の3段階で分かりやすく表示。
  - **鑑賞ポイント：** 「ここを聴くとより楽しめる」という専門家視点のヒントを提供。
  - **タグ付け：** ジャンルや雰囲気（人情噺、滑稽噺、怪談など）を視覚化。

## 5. デザイン・ビジュアル戦略
- **マテリアル感：** 和紙や墨を感じさせる配色（Paper/Ink）を採用。
- **タイポグラフィ：** 信頼感と情緒を伝えるため、明朝体（Noto Serif JP）をベースとしたデザイン。
- **モダンな操作感：** モーションアニメーション（motion/react）を活用し、伝統芸能の中にも「現代の心地よさ」を感じさせるUI/UX。

## 6. 技術スタック
- **Frontend:** React, Vite, Tailwind CSS (v4)
- **Animation:** motion/react
- **AI Engine:** Google Gemini API (Gemini 2.0 Flash)
- **Type Safety:** TypeScript

## 7. 今後の展望・ロードマップ
- **演者連携：** 特定の寄席や落語家のスケジュールと連携。
- **多言語対応：** インバウンド向けに英語・多言語での演目解説を提供。
- **音声プレビュー：** AIによる演目の導入部分の読み上げ機能。
- **マイリスト機能：** 気に入った演目を保存できる機能。
